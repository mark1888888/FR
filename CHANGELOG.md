# FlowRich 更新記錄

## v1.8.0 — 2026-04-17

### 新功能
- **資產總覽改用「個人財務三表」架構**：
  - 第 1 區「資產負債表（存量）」：淨資產 · 流動資產 · 非流動資產 · 總負債
  - 第 2 區「收支儲蓄表（本月流量）」：收入 · 支出 · 儲蓄（淨收支）
  - 第 3 區「財務健康指標」：負債比率、儲蓄率、緊急預備金月數（含綠/橘/紅三級健康等級標示）
- **資產分類改以流動性區分**：
  - 流動資產 = 現金 + 銀行存款 + 應收款
  - 非流動資產 = 投資 + 不動產現值 + 動產現值
- **新增 3 項財務健康指標**（參考標準理財建議）：
  - 負債比率：`總負債 ÷ 總資產`，建議 < 40%
  - 儲蓄率：`本月儲蓄 ÷ 本月收入`，建議 ≥ 20%，至少 10%
  - 緊急預備金：`流動資產 ÷ 本月支出`，建議 3–6 個月

### 改善
- **移除應收/應付款的「帳戶」欄位與綁定功能**：
  - 明細表格移除帳戶欄，資產分析頁的追蹤表改顯示「應收/應付日期」
  - 新增/編輯 Modal 移除收款/付款帳戶下拉
  - `markReceivablePaid()` 不再自動調整帳戶餘額，改為單純標記狀態
  - 若需同時記錄金流，請另至收入/支出頁新增紀錄

### 修正
- **「+ 新增應收款」按鈕在應付款分頁沒切換文字**：切到應付款分頁時按鈕正確顯示為「+ 新增應付款」

### 文件
- 更新 `FORMULAS.md` 為 v1.8.0，加入三表架構、健康指標公式、健康等級閾值表

## v1.7.3 — 2026-04-17

### 改善
- **資產總覽版面重新整理**：
  - 拆成「資產狀況」與「本月收支」兩區，加入分區標題
  - 上區 4 張卡：**淨資產**、**正資產**、**未收款**、**總負債**，彼此金額互不重疊（淨資產 = 正資產 + 未收款 − 總負債）
  - 下區 3 張卡：收入、支出、淨收支
  - 移除原本「總資產（含未收款）」+「未收款」並列的視覺重複
  - 新增「總負債」合併卡（應付款 + 信用卡），並在副標顯示拆分

### 修正
- **淨資產公式跨頁不一致 bug**：資產分析頁原本 `淨資產 = 正資產 − 負債`，漏算未收款與應付款；現與資產總覽統一為 `正資產 + 未收款 − 應付款 − 負債`
- **股價 API 改用 CORS Proxy 代理**：
  - 原本直接呼叫 `mis.twse.com.tw` 會被瀏覽器 CORS 擋下，導致股價常常抓不到
  - 新增 `fetchTWSE()` 輔助函式，依序嘗試 `corsproxy.io`、`allorigins.win`、`codetabs.com` 三個公開代理
  - 任一代理成功即回傳，保留 TWSE 原始即時報價來源
  - 影響範圍：追蹤股票清單（`loadStocks`）、投資組合即時市值（`renderPortfolio`）

### 文件
- 新增 `FORMULAS.md`：完整列出所有金額、統計、公式與資料欄位定義，作為前後端計算的唯一真實來源

## v1.4.0 — 2026-04-16

### 新功能
- **表格排序功能**：收入/支出列表的欄位標題可點擊排序（日期、類別、說明、支付對象、使用對象、金額、幣別、支付方式），再次點擊切換升序/降序，標題旁顯示 ▲▼ 排序指示
- **子類別系統**：收支類別支援建立子類別（如「教育 > 學費」「教育 > 午餐費」等）
  - 設定頁面每個大分類旁新增「＋」按鈕，可展開輸入框新增/刪除子類別
  - 新增收支時的類別下拉選單顯示大分類與縮排子類別
  - 資料儲存格式 `"大分類 > 子分類"`，表格中以小字標籤顯示
  - 分析圖表（圓餅圖）以大分類統計，方便總覽
  - 資料模型新增 `subCategories` 欄位，向下相容既有資料
- **收支分析新增支付對象與使用對象分佈圓餅圖**：合併收入+支出統計，空白值歸類為「其他」
  - 新增 `groupByField()` 通用分組函式
  - `drawPie()` 支援 `noIcon` 參數，對象圖表不顯示類別圖示

### 改善
- **股票追蹤 API 改為 TWSE 臺灣證交所**：
  - 移除 Yahoo Finance API 依賴，改為完全使用 `mis.twse.com.tw` 作為唯一資料來源
  - 一次性同時查詢上市（tse）與上櫃（otc），自動去重
  - 個別 fallback 查詢確保不漏掉任何股票
  - 提取 `parseTWSE()` 和 `renderCards()` 輔助函式，消除重複程式碼
- **OCR 帳單掃描大幅強化**：
  - 影像前處理：灰階化 + 對比增強 + Otsu 自適應二值化（取代固定閾值），自動適應不同亮度帳單；大圖自動縮放
  - 文字清理 `_cleanOcrText()`：移除 CJK 字元間空格（臺 北 市 → 臺北市）、合併被空白切開的數字序列、修正日期斜線空格、全形→半形轉換、常見 OCR 數字誤判修正（O→0, l/I→1）、CJK 與英數間不必要空格移除
  - 金額辨識強化：支援「元」「圓」後綴、更積極的字母→數字修正、過濾短數字避免與日期混淆
  - Tesseract 參數優化：`preserve_interword_spaces: '0'`、`SINGLE_BLOCK` 頁面分段模式
- **民國年日期支援**：`_parseDateStr()` 新增 2-3 位數年份辨識（如 `115/04/13` → `2026-04-13`），`_extractDates()` 新增對應正則

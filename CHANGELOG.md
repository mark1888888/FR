# FlowRich 更新記錄

## v1.8.6 — 2026-04-17

### 新功能

#### A. 信用卡帳單掃描前先選信用卡
- 開啟「掃描帳單」時，最上面新增信用卡下拉選單
- 若尚無信用卡帳戶，顯示提示並附「去新增」按鈕直連資產管理
- 解析後的所有項目預設套用選定的信用卡，亦可於每筆手動調整

#### B. 專案式標籤（跨月跨類別的損益追蹤）
- 新導航項「財務規劃 → 專案標籤」
- 可建立專案（如裝修新房、京都旅行、備孕計畫）：名稱、Emoji、起訖日、預算、幣別、備註
- 收入/支出 Modal 新增「專案標籤」下拉，可貼到任何一筆交易
- 專案卡顯示：淨損益、收入/支出小計、筆數、預算達成百分比（> 100% 變紅）
- 刪除專案不會刪交易，只把標籤清掉

#### C. 定期收支 + 生存天數 + 現金流預判
- 新增「定期收支項目」資料結構：類型（收入/支出）、名稱、金額、幣別、分類、扣款日（1-28）、對應帳戶、啟用旗標
- 每月 1 號後若有項目 `lastGenerated` 不是本月，頂部顯示「本月還有 N 筆待建立」橫幅
- 按下「立即產生本月」會依每筆設定的日期建立實際收入/支出記錄，並調整帳戶餘額
- 資產總覽的財務健康指標新增「生存天數」卡：`快速支配資產 ÷ 近 3 月平均月支出 × 30`；≥180 天健康、90-180 尚可、<90 警訊

#### D. 儲蓄目標（含旅遊預算）
- 新增儲蓄目標資料結構：名稱、Emoji、目標金額、幣別、截止日、綁定專案（選填）
- 目標卡顯示：已累計（專案淨收入）/ 目標、進度條、按本月結餘需幾個月達成；若預估超過截止日會以紅色警告
- 旅遊預算可透過「綁定專案」+「截止日」實現（例：建立專案「京都旅行」+ 目標「60,000 TWD｜2026-12-31」）

#### E. 台灣綜合所得稅預估
- 新增稅務預估頁，依 2025 年度稅率級距（5%/12%/20%/30%/40%）試算
- 自動從收入頁彙總：薪資類 → 薪資所得；其他類 → 其他所得；投資收益類 → 股息
- 顯示：今年累計收入、課稅所得淨額、年稅額、每月建議預備金、有效稅率
- 扣除項預設含免稅額 97,000 + 標準扣除額 131,000 + 薪資所得特別扣除額上限 218,000

### 資料結構變更
- `incomes[*].projectId` / `expenses[*].projectId`：選填欄位，貼專案標籤
- `DB.projects` / `DB.recurring` / `DB.savingsGoals` / `DB.taxProfile`：新增四個陣列/物件

## v1.8.4 — 2026-04-17

### 改善
- **投資改歸流動資產**：
  - 資產負債表分類調整：`流動資產 = 現金 + 銀行存款 + 投資 + 應收款`、`非流動資產 = 不動產 + 動產`
  - 理由：上市股票/ETF/基金 T+2 即可變現，嚴格會計上亦屬流動資產，個人理財反映實際變現能力更準
  - 緊急預備金月數自動變得更樂觀（分母 liquid 多了投資部位）
  - 「快速支配資產」卡片公式不變（銀行+現金+投資），意義更新為「流動資產扣除應收款」，可從財務健康指標卡的副標看出
  - 卡片副標同步更新

## v1.8.3 — 2026-04-17

### 新功能
- **應收/應付款連動帳戶自動記帳**：
  - Modal 新增/編輯應收/應付款時可選擇「收款帳戶」/「付款帳戶」（僅列出銀行、現金、投資三類可實際收付的帳戶）

  - 自動檢查並新增「應收款入帳」/「應付款支付」兩個類別到使用者類別清單（若尚未存在）
  - 未指定帳戶時仍可「僅更新狀態」，系統會跳提示告知不會動到餘額
- **刪除已結清應收/應付款時連動還原**：若刪除的記錄已經「標記已收/已付」並連結收入/支出紀錄，會同步刪除該筆收/支並回補帳戶餘額，避免資料不一致
- **明細表格新增「連動帳戶」欄**：直接在表格看到這筆應收/應付款結清時會影響哪個帳戶；未指定則顯示灰色「未指定」

## v1.8.0 — 2026-04-17

### 新功能
- **資產總覽改用「個人財務三表」架構**：
  - 第 1 區「資產負債表（存量）」：淨資產 · 流動資產 · 非流動資產 · 總負債
  - 第 2 區「收支儲蓄表（本月流量）」：收入 · 支出 · 儲蓄（淨收支）
  - 第 3 區「財務健康指標」：負債比率、儲蓄率、緊急預備金月數（含綠/橘/紅三級健康等級標示）
- **資產分類改以流動性區分**：
  - 流動資產 = 現金 + 銀行存款 + 應收款
  - 非流動資產 = 投資 + 不動產現值 + 動產現值
- **新增 3 項財務健康指標**（參考標準理財建議）：
  - 負債比率：`總負債 ÷ 總資產`，建議 < 40%
  - 儲蓄率：`本月儲蓄 ÷ 本月收入`，建議 ≥ 20%，至少 10%
  - 緊急預備金：`流動資產 ÷ 本月支出`，建議 3–6 個月

### 改善
- **移除應收/應付款的「帳戶」欄位與綁定功能**：
  - 明細表格移除帳戶欄，資產分析頁的追蹤表改顯示「應收/應付日期」
  - 新增/編輯 Modal 移除收款/付款帳戶下拉
  - `markReceivablePaid()` 不再自動調整帳戶餘額，改為單純標記狀態
  - 若需同時記錄金流，請另至收入/支出頁新增紀錄

### 修正
- **「+ 新增應收款」按鈕在應付款分頁沒切換文字**：切到應付款分頁時按鈕正確顯示為「+ 新增應付款」

### 文件
- 更新 `FORMULAS.md` 為 v1.8.0，加入三表架構、健康指標公式、健康等級閾值表

## v1.7.3 — 2026-04-17

### 改善
- **資產總覽版面重新整理**：
  - 拆成「資產狀況」與「本月收支」兩區，加入分區標題
  - 上區 4 張卡：**淨資產**、**正資產**、**未收款**、**總負債**，彼此金額互不重疊（淨資產 = 正資產 + 未收款 − 總負債）

- **淨資產公式跨頁不一致 bug**：資產分析頁原本 `淨資產 = 正資產 − 負債`，漏算未收款與應付款；現與資產總覽統一為 `正資產 + 未收款 − 應付款 − 負債`

  - 任一代理成功即回傳，保留 TWSE 原始即時報價來源
  - 影響範圍：追蹤股票清單（`loadStocks`）、投資組合即時市值（`renderPortfolio`）

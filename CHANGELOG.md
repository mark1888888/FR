## v1.11.2 — 2026-04-17 · 健康指標置頂 + 動產價值自動化

### 改變 A：資產總覽順序調整
- 原順序：資產負債表 → 收支儲蓄表 → 財務健康指標
- **新順序**：**財務健康指標（最上）** → 資產負債表 → 收支儲蓄表
- 一打開 App 第一眼看到的是健康度評分（負債比、儲蓄率、緊急預備金、快速支配、生存天數），有重點提示意義

### 新功能 B：動產「價值變動方式」
動產 Modal 新增「價值變動方式」下拉，三種模式：

#### 🔧 手動更新（預設）
原行為不變 — 使用者自行輸入 currentValue。

#### 🔻 折舊型（系統自動按月扣減）
適用汽車、機車等會隨時間貶值的動產。
- 輸入「年折舊率 %」（汽車常用 15–20）
- 輸入「折舊起算日」（預設 = 購入日期）
- **公式**：`當前價值 = 購入價 × (1 - 年折舊率)^年數`
  - 採定率遞減法，前期折舊較快
  - 範例：80 萬汽車、15%/年、3 年後 ≈ 49.1 萬、5 年後 ≈ 35.5 萬
- 啟動時自動套用、儲存後也會立即重算
- 列表顯示「🔻 年折舊 15%」小標

#### 🔗 保值型（連動國際金價/行情）
適用黃金、白銀、名錶（手動更新）等。
- 連動標的：🟡 黃金 / ⚪ 白銀 / ⚫ 鉑金 / 🔘 鈀金
- 持有單位（盎司 oz）
- **公式**：`當前價值 = 持有盎司 × metals.dev 現價 × 幣別換算`
- 金屬價存在 `DB.lastMetalPrices` 快取（投資情報頁每次開啟會刷新）
- 抓到新價格時自動重算所有保值型動產
- 留空 `持有單位` 則只做手動更新，不連動
- 列表顯示「🔗 黃金 × 5 oz」小標

### 資料結構新增欄位（於 `vehicles[*]`）
- `valuationMode`: 'manual' | 'depreciation' | 'market'
- `depreciationRate`: 年折舊率 %（depreciation 用）
- `depreciationStart`: 折舊起算日
- `marketLink`: 連動金屬代號（market 用）
- `marketUnits`: 持有盎司數
- `DB.lastMetalPrices`: 金屬價快取 { gold, silver, platinum, palladium, _updatedAt }

### 自動觸發時機
1. **登入時**：讀取 DB 快取立即套用一次
2. **儲存動產時**：立即重算該筆
3. **投資情報頁開啟時**：fetch 最新金屬價 → 回寫 DB → 重算所有保值型資產

### 使用範例
- 建立「2023 Toyota Camry」動產，購入價 90 萬、15%/年折舊 → 今年自動變成約 76.5 萬、明年 65 萬
- 建立「黃金條塊」動產，連動黃金、5.0 oz → 立刻顯示 ≈ NT$378,000（若金價 2,400 USD/oz × 31.5 匯率）
- 建立「勞力士 Submariner」，保值型但留空盎司 → 手動更新市價，不連動自動計算

## v1.11.0 — 2026-04-17 · 修投資市值首次載入

### 修正
**問題**：資產總覽的「流動資產」副標顯示的投資金額是**成本**（如 NT$989,890）而不是**市值**（如 NT$808,100）。

**Root cause**：v1.10.4 為了省 API 請求，改成「只在開盤時段 fetch 股價」。但這對「從沒在開盤時打開過 App」的使用者造成副作用：
- 快取 `DB.lastPortfolioPrices` 從未被寫入
- `portfolioMarketValue()` 沒價格資料可用，fallback 成成本價
- 非開盤時段又不 fetch，永遠顯示成本

### 修法
啟動時判斷快取狀態：
- **無快取**（新使用者或從未抓成功）→ 不管開不開盤**必 fetch 一次**
- **有快取 + 開盤中** → fetch 刷新到最新
- **有快取 + 非開盤** → 用快取不發請求（省 API）

這樣確保第一次打開 App（或 cache 被清掉後）會強制抓一次市值並存入快取，之後才進入「開盤才刷新」的節流模式。

### 邏輯驗證
| 情境 | 是否 fetch |
|------|----------|
| 無快取 + 非開盤 | ✓ |
| 無快取 + 開盤 | ✓ |
| 有快取 + 非開盤 | ✗ |
| 有快取 + 開盤 | ✓ |

## v1.10.4 — 2026-04-17 · 觸控修正 + 數字動畫 + 股價快取

### 修正 — 手機觸控 hover 卡住
- 所有 hover lift 效果（卡片上浮、按鈕光暈、nav 縮排、icon 放大）在**觸控裝置**會「黏住」，因為觸控後 `:hover` 不會自動清除
- 解法：在 CSS 尾端加 `@media (hover: none) { ... }` 將所有 transform / box-shadow / border-color 增量**回復為預設值**
- 滑鼠使用者（`hover: hover`）完整體驗保留；觸控使用者乾淨利落，按完就釋放
- `:active` 的短暫縮放回饋保留（觸控也會觸發）

### 新功能 — 數字動畫
- 資產總覽卡片的金額在**首次登入**與**切換回 dashboard**時會從 0 跳動到目標值
- 用 `requestAnimationFrame` + easeOutCubic 曲線，900ms 完成
- 自動保留原有格式：千位逗號、幣別前綴、正負符號、小數位
- 卡片間 stagger 80ms 錯開，像依序點亮
- 尊重 `prefers-reduced-motion`（使用者系統設定關閉動畫 → 跳過不跑）
- 同頁 re-render（例如新增一筆支出）不會重播動畫

### 改善 — 投資市值快取
**問題**：登入後第一眼看到的「投資」金額是**成本價**，要等股價 fetch 完才跳到市值，體驗不連貫。而且就算非開盤時段也會發不必要的 API 請求。

**修法**：
- 新增 `DB.lastPortfolioPrices` 作為**持久化快取**，每次成功 fetch 後寫入並同步雲端
- `portfolioMarketValue()` 優先順序：**本次 session 即時價 → DB 快取 → 成本價 fallback**
- 登入時立即顯示上次儲存的市值（通常就是合理數字）
- 只在**開盤時段**（週一～五 09:00–13:30）才 fetch 新價格
- 非開盤時間不打 API，純看快取
- 投資組合明細表格的「現價」與「市值」欄位也套用快取邏輯

### 技術細節
- `_animateNumberText(el, opts)` — 通用數字動畫工具，可套到其他卡
- 手機觸控偵測：`@media (hover: none) and (pointer: coarse)` — 精確地針對「純觸控」裝置
- 股價快取同步時機：在 `fetchPortfolioPrices()` 成功後 `save()`（會上傳 Supabase）

## v1.9.5 — 2026-04-17 · 手機右側空白 bug 修正

### 關鍵修正
**手機版（含桌面切換過的）右側空出約 30% 空白**
- Root cause：`enterApp()` 把 `.app` 設為 `display:flex`，但 `.sidebar` 和 `.bottom-nav` 是 `position:fixed` 不佔 flex slot，flex 容器內只剩 `.main` 一個 item
- `.main` 作為 flex item 預設 `flex:0 1 auto`，只會取「內容寬度」，不會自動填滿剩餘空間
- 結果：手機上 sidebar 寬度 0，但 `.main` 還是縮到 content 寬度，右側出現大片空白

### 修法
1. `enterApp()` 的 `display:'flex'` → `'block'`：sidebar 是 fixed，不需要 flex 容器
2. `.main` 加 `overflow-x:hidden` 防止意外溢出
3. `html, body { overflow-x:hidden; max-width:100vw }` 全域保險
4. 手機版 `.main { width:100%; max-width:100vw }`、`.page { width:100% }` 明確設定

登出時的 `display:'none'` 保持不變，不影響該邏輯。
